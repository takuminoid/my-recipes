## 3. API Routes

- [x] 3.1 `GET /api/recipes` を実装する（全件取得・`?q=` による料理名検索・`cooked_at` 降順ソート）
- [x] 3.2 `POST /api/recipes` を実装する（レシピ + 材料リストを同一トランザクションで新規作成）
- [x] 3.3 `GET /api/recipes/[id]` を実装する（レシピ + 材料リストを結合して1件取得）
- [x] 3.4 `PUT /api/recipes/[id]` を実装する（レシピ更新 + 材料を削除→再挿入でトランザクション更新）
- [x] 3.5 `DELETE /api/recipes/[id]` を実装する（CASCADE により材料も自動削除）
